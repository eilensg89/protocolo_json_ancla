Protocolo JSON Ancla

Este documento corrige el flujo real del sistema ESG. Aquí no se copia una imagen. Se trabaja con JSON y bloqueos que la IA puede leer con jerarquía clara: escenario, sujeto y guía. Primero se extrae el JSON exacto de una referencia. Después se sustituye el sujeto con bloqueos de identidad. Luego se hacen solo los cambios pedidos. Al final se genera la imagen conservando pose, escena, estructura visual y calidad 8K sin ruido.

Qué aplica el sistema ESG

El protocolo funciona separando la imagen en tres capas. Esa separación evita que la IA mezcle identidad, escena y cambios pedidos.

Escenario

Es la estructura de la imagen: encuadre, pose, composición, luz, fondo, ambiente, objetos y atmósfera. Esta capa nace de la referencia Pinterest y normalmente se conserva.

Sujeto

Es la persona o figura principal. Aquí se sustituye la persona del JSON 1 por el rostro y cuerpo base del usuario, manteniendo la estructura visual del escenario.

Guía

Son las instrucciones de control: qué sí cambia, qué no cambia, qué colores usar, qué estilo de ropa aplicar y qué escenario conservar. La guía limita a la IA.

Resumen práctico: Escenario = lo que se copia estructuralmente. Sujeto = a quién se reemplaza. Guía = qué cambios autorizas y qué elementos deben permanecer intactos.

Flujo corregido

Este es el orden real para que el sistema funcione bien y no altere cosas que no pediste.

1

Referencia Pinterest → JSON 1

Se sube una sola referencia de Pinterest. Se extrae el JSON exacto de la imagen. Ese JSON debe contener la estructura visual detectada: pose, luz, composición, fondo, ropa, ambiente y estilo. Este primer JSON no cambia todavía el sujeto.

2

Rostro + cuerpo base → JSON 2

Se suben dos imágenes más: una foto del rostro y una foto del cuerpo base. Luego se toma el JSON 1 y se pide sustituir las características del sujeto original por las de esas dos referencias. El resultado es el JSON 2.

3

Cambios selectivos sobre JSON 2

Después se toma el JSON 2 y se piden solo los cambios específicos: color de ropa, estilo de ropa, fondo y ambiente, solo si el usuario lo pide. La pose, la estructura y los elementos de la imagen se copian exactamente. Si el usuario no pide cambiar el escenario, el escenario no se puede cambiar.

4

Prompt final de generación

Con el JSON corregido y las imágenes ordenadas, se crea el prompt final: Imagen A = rostro, Imagen B = cuerpo base, Imagen C = referencia de escena. El prompt final indica a la IA qué referencias usar para el sujeto y cuál usar para escena, pose, luz y atmósfera.

Reglas que no se rompen

Lo que se conserva

Pose, encuadre, composición, dirección de luz, distribución de elementos y escenario original, salvo que el usuario pida un cambio puntual.

Lo que sí puede cambiar

Identidad del sujeto, color de ropa, estilo de ropa, fondo y ambiente, pero solo en la medida exacta de lo que el usuario especifique.

Regla central: si el usuario no pide cambio de escenario, el escenario se conserva. El protocolo no autoriza cambios “creativos” no solicitados.

Bloqueos que la IA sí entiende

El protocolo no depende de pedir “parecido” en lenguaje suelto. Usa llaves de bloqueo dentro del JSON para separar identidad, cuerpo, escena y calidad.

IDENTITY_LOCK

Rostro, cejas, ojos, nariz, labios, tono de piel, cabello, edad percibida, expresión base e identidad reconocible. Tiene prioridad absoluta sobre la referencia Pinterest.

BODY_LOCK

Proporciones corporales, complexión, altura percibida, postura base, volumen corporal y presencia física. La escena no puede cambiar el cuerpo.

SCENE_LOCK

Pose, encuadre, cámara, luz, fondo, ambiente, objetos y composición de la referencia. Solo cambia si el usuario lo autoriza.

QUALITY_LOCK

8K, máxima nitidez al hacer zoom, microdetalle limpio, piel natural, textura real, cero ruido y cero grano salvo que el usuario lo pida.

Jerarquía oficial: IDENTITY_LOCK + BODY_LOCK mandan sobre el sujeto. SCENE_LOCK manda sobre la escena. CHANGE_REQUEST solo autoriza cambios específicos. QUALITY_LOCK protege el resultado final.

Completa los campos

Estos campos alimentan automáticamente el prompt para crear el JSON 2, el prompt de ajustes sobre el JSON 2 y el prompt final de generación.

Prompts corregidos

Los prompts ya siguen el flujo correcto. El prompt 2 deja el espacio para pegar el JSON 1. El prompt 3 trabaja solo sobre el JSON 2. El prompt final te dice el orden exacto de las imágenes.

Prompt 1 — Extraer JSON 1 de la referencia Pinterest
Voy a subir una imagen de referencia de Pinterest.

Extrae el JSON visual exacto de esta imagen.

No quiero una descripción superficial ni un prompt narrativo.
Quiero un JSON técnico, limpio y reutilizable que la IA pueda entender como estructura visual.

Entrega el resultado como JSON 1 con estas llaves obligatorias:

{
  "REFERENCE_TYPE": "Pinterest visual reference",
  "SCENE_LOCK": {
    "composition": "",
    "camera_angle": "",
    "framing": "",
    "pose": "",
    "lighting": "",
    "background": "",
    "environment": "",
    "props": "",
    "object_positions": "",
    "color_palette": "",
    "visual_style": "",
    "mood": ""
  },
  "ORIGINAL_SUBJECT_TO_REPLACE": {
    "face": "",
    "body": "",
    "hair": "",
    "outfit": "",
    "expression": ""
  },
  "DO_NOT_COPY": [
    "original person identity",
    "original facial features",
    "original body identity"
  ],
  "KEEP_EXACT": [
    "pose",
    "composition",
    "scene structure",
    "lighting direction",
    "camera framing",
    "environment",
    "props and object placement"
  ]
}

Regla:
Este JSON 1 solo extrae escenario, pose, composición, luz, objetos y estilo.
Todavía no sustituyas el sujeto.
Todavía no hagas cambios creativos.

Cómo aplicar el protocolo sin romperlo

        

          
1
Extrae JSON 1
Siempre desde la referencia Pinterest, sin mezclar todavía rostro ni cuerpo del usuario.

          
2
Crea JSON 2
Sustituye el sujeto original usando imagen A para el rostro e imagen B para el cuerpo base.

          
3
Ajusta solo lo pedido
Haz cambios controlados en ropa, estilo, fondo o ambiente sobre el JSON 2. No cambies escenario si el usuario no lo pidió.

          
4
Genera con A, B, C + JSON final
El sujeto sale de A y B. La escena sale de C. La guía final sale del JSON ajustado y las reglas del usuario.
